Detection and Correction of Photosensitive Epilepsy Triggers in Videos

A system for identifying and mitigating epileptic seizure triggers in video content

Hacettepe University - Computer Engineering Department

Abstract

Photosensitive epilepsy affects approximately 3% of people with epilepsy, with seizures triggered by specific visual patterns including rapid flashing lights, high contrast patterns, and certain color combinations. This research presents a comprehensive system for detecting and correcting epileptic triggers in video content using advanced deep learning techniques.

Our approach employs a transformer-based architecture combined with EfficientNet for robust feature extraction from video sequences. The system processes 30-frame sliding windows with temporal attention mechanisms to identify potentially harmful visual patterns. Upon detection, an adaptive correction algorithm applies context-aware modifications including overlay opacity adjustment, temporal smoothing, and intelligent color intensity reduction while preserving content integrity.

Experimental validation using the Photosensitive Epilepsy Analysis Tool (PEAT) demonstrates significant effectiveness in neutralizing dangerous patterns while maintaining visual quality. The system represents a crucial step toward making digital media more accessible and safer for individuals with photosensitive epilepsy.

🎯 Key Contributions

  • ✅ Novel Detection Framework: First implementation of transformer-based temporal modeling for photosensitive trigger detection in videos
  • ✅ Adaptive Correction Algorithm: Context-aware modification system that preserves content quality while ensuring safety
  • ✅ Comprehensive Evaluation: Extensive testing using PEAT validation and real-world video datasets
  • ✅ Accessibility Impact: Practical solution for making digital media safer for photosensitive individuals

🎯 Performance Metrics

  • Accuracy: 88.3% on test set
  • Precision: 90.7% (low false positives)
  • Recall: 94.2% (high sensitivity)
  • F1-Score: 92.4% balanced performance

Processing Pipeline

Our detection and correction pipeline consists of six main stages, each optimized for accuracy and efficiency in trigger identification while maintaining video quality and processing speed.

1

Video Input & Preprocessing

Frame Extraction: Videos are processed using 30-frame sliding windows with 15-frame stride, ensuring 50% overlap for comprehensive temporal coverage without missing critical transitions.

Preprocessing: BGR to RGB conversion, resize to 224×224 pixels, and ImageNet normalization for optimal feature extraction compatibility.

2

Feature Extraction

CNN Backbone: EfficientNet-B0 extracts 1280-dimensional spatial features from each frame, capturing both local patterns and global scene characteristics.

Feature Processing: Linear projection layer reduces dimensionality to 1024 for efficient transformer processing while preserving critical visual information.

3

Temporal Modeling

Transformer Architecture: 3-layer encoder with 4-head multi-attention mechanism models complex temporal dependencies across the 30-frame sequence.

Positional Encoding: Learnable embeddings provide crucial temporal order information for accurate pattern recognition.

4

Trigger Classification

Binary Decision: Sigmoid classifier with optimized threshold (0.7) balances sensitivity and specificity for reliable trigger detection.

Confidence Scoring: Probability outputs enable fine-tuned decision making and uncertainty quantification.

5

Adaptive Correction

Context-Aware Modifications: Overlay opacity (0.3-0.7), temporal smoothing, and selective color channel intensity reduction based on trigger severity.

Content Preservation: Intelligent algorithms maintain visual coherence while ensuring PEAT compliance.

6

Quality Validation

PEAT Verification: Automated validation using Photosensitive Epilepsy Analysis Tool ensures corrected content meets safety standards.

Quality Metrics: SSIM and PSNR evaluation maintains acceptable visual fidelity (>0.85 SSIM threshold).

Detection Model Architecture

Our detection model employs a hybrid CNN-Transformer architecture specifically designed for identifying photosensitive epilepsy triggers in video sequences. The system processes 30-frame windows through a sophisticated temporal modeling approach.

🎬 Video Input Processing

Sliding Window: 30-frame sequences extracted with 15-frame stride (50% overlap)

Frame Preprocessing: BGR→RGB conversion, resize to 224×224, ImageNet normalization

Batch Reshaping: Input tensor (B, 30, 3, 224, 224) → (B×30, 3, 224, 224)

Window Parameters: Batch size 4, ensuring comprehensive temporal coverage

🧠 CNN Feature Extraction

Backbone Network: EfficientNet-B0 (ImageNet pretrained) for spatial feature extraction

Feature Dimensions: Each frame produces 1280-dimensional feature vector

Linear Projection: Features projected to 1024 dimensions for transformer compatibility

Dropout Regularization: 0.3 dropout rate applied after projection layer

📍 Temporal Encoding

Sequence Reshape: Features reshaped from (B×30, 1024) to (B, 30, 1024)

Positional Encoding: Learnable embeddings (1, 30, 1024) added to frame features

Initialization: Positional encodings initialized with normal distribution (std=0.02)

Temporal Awareness: Explicit frame order information for sequence modeling

🔄 Transformer Encoder

Architecture: 3 transformer encoder layers with multi-head self-attention

Attention Configuration: 4 attention heads, 1024 model dimension

Feed-Forward Network: Hidden dimension 4096 with ReLU activation

Normalization: Layer normalization with residual connections

🎯 Classification Pipeline

Global Pooling: Mean pooling across temporal dimension (B, 30, 1024) → (B, 1024)

MLP Architecture: Three-layer classifier (1024→256→64→1)

Activation Functions: ReLU activations with 0.3 dropout between layers

Output: Binary classification with sigmoid activation for trigger detection

Detection Algorithm

🎬 Video Processing Pipeline

Input Format: Video frames extracted in 30-frame sliding windows with 15-frame stride (50% overlap)

Preprocessing: BGR→RGB conversion, resize to 224×224, ImageNet normalization (μ=[0.485, 0.456, 0.406], σ=[0.229, 0.224, 0.225])

Batch Processing: Frames reshaped to (B×30, 3, 224, 224) for efficient parallel processing through CNN backbone

Memory Optimization: Gradient checkpointing and mixed precision training for handling large temporal sequences

🧠 Feature Extraction Module

CNN Backbone: EfficientNet-B0 (ImageNet pretrained) extracts 1280-dimensional spatial features per frame

Feature Projection: Linear layer (1280→1024) with ReLU activation projects features for transformer compatibility

Positional Encoding: Learnable 1024-dimensional embeddings provide temporal frame order information

Dropout Regularization: 0.1 dropout rate applied after projection to prevent overfitting

🔄 Temporal Transformer

Architecture: 3 encoder layers with 4-head self-attention (d_model=1024, d_ff=4096)

Self-Attention: Captures complex temporal dependencies across 30-frame sequences for flicker pattern detection

Layer Normalization: Pre-norm configuration with residual connections for stable training

Global Pooling: Mean pooling across temporal dimension produces final 1024-dimensional sequence representation

🎯 Classification Head

Architecture: Two-layer MLP (1024→512→1) with ReLU activation and dropout (0.3)

Output: Single sigmoid neuron for binary trigger classification (safe vs. dangerous)

Loss Function: Binary cross-entropy with class weights (1:3) to handle class imbalance

Decision Threshold: Optimized threshold (τ=0.7) balances precision and recall for safety-critical application

⚙️ Training Configuration

Optimizer: AdamW with learning rate 1e-4, weight decay 1e-2, and cosine annealing schedule

Data Augmentation: Random horizontal flip, color jitter (brightness±0.2, contrast±0.2), and temporal jittering

Batch Size: 16 sequences per batch (480 total frames) with gradient accumulation for effective batch size of 64

Validation: Early stopping based on F1-score with patience of 10 epochs

Correction Algorithm

Upon detection of photosensitive triggers, our adaptive correction algorithm applies context-aware modifications to neutralize harmful patterns while preserving visual quality and content integrity.

🎭 Adaptive Overlay Correction

  • Dynamic Opacity: 0.3-0.7 based on luminance analysis
  • Brightness Calculation: YUV color space conversion
  • Overlay Generation: Gaussian blur (σ=5) + alpha blending
  • Edge Enhancement: Sobel operator for detail preservation

📊 Flicker Detection & Smoothing

  • Frequency Analysis: FFT-based 15-25Hz detection
  • Temporal Buffer: 7-frame sliding window analysis
  • Gaussian Filter: σ=1.2 temporal kernel smoothing
  • Motion Compensation: Lucas-Kanade optical flow

🌈 Intelligent Color Reduction

  • Red Channel: 15-35% logarithmic intensity scaling
  • Blue Channel: 10-28% selective dimming algorithm
  • HSV Preservation: Hue locked, saturation max 80%
  • Perceptual Model: CIE L*a*b* color space weighting
